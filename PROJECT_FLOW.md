# Smart Hospital Information System — Project Flow

This document explains the current project flow, where the main files are located, and what they contain.

## 1. Project entry points

### `src/start.ts`
- Registers global middleware for the app.
- Uses `@tanstack/react-start` to create a start instance.
- Attaches Supabase authentication middleware and error middleware.
- This is the server-side startup entry for the full app.

### `src/server.ts`
- The server-side entry point exported as a default fetch handler.
- Imports TanStack React Start server entry and normalizes SSR errors.
- Renders a friendly error page if the server render fails.

### `src/router.tsx`
- Creates the main router using `@tanstack/react-router`.
- Loads the generated route tree from `src/routeTree.gen.ts`.
- Configures `QueryClient` for React Query.

## 2. Routing and page structure

### `src/routeTree.gen.ts`
- Auto-generated route tree for the app.
- Maps file-based routes from `src/routes/` into the router.
- Do not edit this file manually.

### `src/routes/__root.tsx`
- Root route and shell for the application.
- Wraps the app in `QueryClientProvider`, `AuthProvider`, and `PatientDataProvider`.
- Handles meta tags, global styles, error handling, and 404 pages.
- Defines the HTML document shell used for all app pages.

### `src/routes/index.tsx`
- Redirects the root path `/` to `/login`.

### `src/routes/login.tsx`
- Parent route for login pages.
- Displays nested login-specific pages via `Outlet`.

### `src/routes/login.index.tsx`
- Role selection landing page.
- Shows buttons for Receptionist, Triage Nurse, Clinician, Lab Technician, and Pharmacist.

### `src/routes/login.*.tsx`
- Individual login pages for each role.
- Use `CredentialPage` to render a login form for the selected role.
- Paths:
  - `/login/clinician`
  - `/login/lab`
  - `/login/pharmacy`
  - `/login/receptionist`
  - `/login/triage`

### `src/components/his/CredentialPage.tsx`
- Shared login page component used by all role login routes.
- Handles sign-in and navigation to the role-specific dashboard.
- Seeds inventory on first login using `ensureInventorySeeded()`.

## 3. Context providers and global state

### `src/context/AuthContext.tsx`
- Provides authentication state and staff profile data.
- Uses Firebase Auth and Firestore to watch the signed-in user and staff document.
- Exposes `signIn`, `signOut`, `isAuthenticated`, `user`, and `staff`.
- Controls role-based auth state for page guards.

### `src/context/PatientDataContext.tsx`
- Provides realtime app data from Firestore collections.
- Loads collections when `isAuthenticated` is true.
- Collections included:
  - `patients`
  - `visits`
  - `payments`
  - `triage_records`
  - `consultations`
  - `lab_requests`
  - `lab_results`
  - `prescriptions`
  - `inventory`
  - `outbound_notifications`
- Exposes helper selectors like `patientById`, `trigeByVisit`, `paymentByVisit`, and `consultationByVisit`.

## 4. Shared UI and helpers

### `src/components/his/shared.tsx`
- Shared UI components and helpers used across dashboards.
- Contains:
  - `RoleGuard` — protects routes based on role and login state.
  - `RoleHeader` — shared header for dashboard pages.
  - `FlowTracker` — live patient flow cards by visit status.
  - `StatCard` — summary card component.
  - `Tabs` — tab navigation UI used inside dashboards.
  - `Empty`, `Spinner`, `Field`, `inputCls` — UI helpers.
  - `useToast` — lightweight toast notification helper.
  - Utility functions: `isToday`, `ageFromDob`, `priorityBadge`, `statusBadge`.

### `src/lib/firestore-helpers.ts`
- Firestore helper utilities and shared app logic.
- Provides:
  - `nextCode()` — generates human-readable codes like `LAB-001` or `RX-001`.
  - `canProceedToService()` — checks payment state and insurance validity.
  - `matchInventoryItem()` — matches medication names to inventory items.
  - `buildDailyReportSummary()` — builds a daily report object.
  - `ensureInventorySeeded()` — seeds default inventory items.
  - Demo staff definitions and sign-in seed logic.

### `src/lib/firebase.ts`
- Firebase client setup for Firestore and Authentication.
- Likely configures and exports `db` and `auth` for the app.

### `src/lib/error-capture.ts` and `src/lib/error-page.ts`
- Error logging and rendering support for server-side failure states.
- `error-capture` tracks exceptions during SSR.
- `error-page` returns HTML for an error response.

### `src/lib/lovable-error-reporting.ts`
- Integrates with Lovable error reporting.
- Used by the root route error boundary in `src/routes/__root.tsx`.

## 6. Role dashboards

Each dashboard route uses `RoleGuard` and shared UI helpers.

### `src/routes/dashboard.clinician.tsx`
- Clinician dashboard and consultation workflow.
- Displays patient queue, consultation form, lab requests, prescription creation, and lab results.
- Creates lab requests, saves consultations, updates visit status, and issues prescriptions.
- Uses Firestore writes and batch updates to create consultations, lab requests, and prescriptions.

### `src/routes/dashboard.lab.tsx`
- Lab technician dashboard.
- Displays pending lab requests, upload results form, and historical results.
- Updates `lab_requests` status when processing and writes `lab_results`.
- Moves visits back to clinician flow when all lab requests for a visit are completed.

### `src/routes/dashboard.pharmacy.tsx`
- Pharmacist dashboard.
- Likely displays prescriptions, dispensing flows, and inventory updates.

### `src/routes/dashboard.receptionist.tsx`
- Receptionist dashboard.
- Likely handles patient registration, billing, and queue management.

### `src/routes/dashboard.triage.tsx`
- Triage nurse dashboard.
- Likely captures vitals, assigns priority, and moves patients into consultation queue.

## 9. High-level flow summary

1. User opens `/` and is redirected to `/login`.
2. `login.index.tsx` lets the user choose a staff role.
3. The selected `/login/{role}` page renders `CredentialPage`.
4. `CredentialPage` signs in via `AuthContext` and seeds inventory.
5. After login, the user is routed to the role-specific dashboard.
6. `__root.tsx` wraps pages with `AuthProvider` and `PatientDataProvider`.
7. `AuthContext` tracks the logged-in Firebase user and role profile.
8. `PatientDataContext` subscribes to Firestore collections for app data.
9. Each dashboard component reads shared data and updates Firestore.
10. Lab requests and results flow between clinician, lab, and pharmacy through Firestore collections.
